## 1. 怎么去设计一个 Agent

设计一个 Agent 前首先要确认你的目标是什么？ 是去取代人还是去增强人。这是两种完全不同的设计逻辑。

如果你去设计一个取代人的 Agent ，我们的目标是 Risk Control。这个是 peek 在访谈中谈到的，这个和 Harness Engineering 思想真的不谋而合。

很多人在 Apr-9-2026 的时间段拿 Claude 最新的模型说事情，说他的能力非常强，可以找到隐藏多年的 BUG，但是实际上，他是 pass k 和 pass^k 的区别



## 2. Ai 时代我们的护城河在哪里

其实很简单，我们思考当前的大模型是怎么出来的？是通过大量数据训练出来的。

所以对于一个人的护城河必然是大模型拿不到的数据，拿不到的数据，缺少的数据就是 ai 没有办法取代我们的地方。


同时对于小公司来讲，我们没有能力去训练大模型，那我们做什么样的 agent 才不会被大模型公司取代呢？

就是数据要造出差异，那些大模型厂商拿不到的数据，就是我们的究极护城河。

如果初始阶段我们拿不到数据怎么办呢？那其实就是记忆，一个用户在我们 Agent 上记忆，是配置。
