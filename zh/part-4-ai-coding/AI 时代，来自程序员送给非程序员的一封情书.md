

因为 ai 的到来，程序员以编程语言为护城河的时代彻底过去了。新时代中，每一个人都会有机会用 AI 编程去做产品，去完成软件的开发。使用自己的专业知识去来打造面向大众的专业 APP，护城河就在于你的专业知识。

## 1. 前期准备
### 1.1 AI 工具的选择

工欲善其事，必先利其器。

我在这里，还是推荐 Claude Code 、Cursor、Codex 等作为编程工具去进行开发。

很多时候，不使用最前沿的编程工具，这是会让小白入门的难度会更加的高。一个问题对于 Claude Code 等工具，很有可能是一句话就解决的问题，而对于绝大多滥竽充数的 AI 编程工具是需要不断的介入，也有可能解决不了。

这些工具，对于国内的朋友，因为诸多的限制，我在这里是推荐使用 Codex、Claude Code 这两个软件。因为他们支持使用第三方大模型供应商去提供服务。

对于第三方大模型供应商的选择，我推荐闲鱼或者是淘宝去搜索中转站相关的服务。还有的话就是支持国产，使用智谱和 Minmax 等相关产品。

在国外产品上，GPT 5.4 我认为能力和 Claude Code 是差不多的，

## 2. 开始开发

## 2.1 描述需求

我们需要严格把握的一点是 AI 是个概率模型，通过你的输入来生成输出。所以尽可能详细的去描述你的需求。

如果你不知道你的需求是什么怎么办？通过让 AI 来反问你。

> 我现在想要开发一款xxx 的产品，现在你需要对相关内容进行反问我，进行澄清整个产品的功能、定位等信息。



## 2.1 复利工程

我觉得 ai coding 比较重要的事情就是 复利工程，什么意思呢，就是将你经常要给 ai 的提示保存下来，形成可以复用的路径。


## 打造自己的Agent 军团

尝试多工作区模式，andrej karpath 说的羡慕龙虾创始人有能力同时处理十几个 agent。

而对于一个项目来讲，可以尝试多工作区，每个工作区是你的项目的不同分支。然后同时开发， commit and push。

最后留给一个主工作区，去做收口，将所有的分支合并。


## Auto Development

auto-research 的翻版，设定一个目标，然后自动循环工作直到完成目标。

个人最佳事件是通过文件通信，并且安排三个角色，developer，reviewer，协调者。
