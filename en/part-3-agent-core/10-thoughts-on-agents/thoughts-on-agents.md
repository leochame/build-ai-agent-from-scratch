# 10. Thoughts on Agents

This chapter will always stay at the end.

It is not meant to be a standard tutorial chapter. It is an evolving record of how I think about agents.

If the earlier chapters ask "how should an agent be built," this chapter asks questions like:

- what an agent really is
- why we need agents at all
- what agents actually change
- what is short-term hype and what is durable value

These questions will not be settled once and for all, so this chapter is intentionally open-ended.

## 1. How Should We Design an Agent?

Before designing an agent, the first question is the goal:

are we trying to replace humans, or augment humans?

Those are very different design directions.

If the goal is replacement, risk control becomes central.

## 2. Where Is the Moat in the AI Era?

Large models are built from large-scale data.

That means one durable moat is data the model makers do not have.

For individuals, that may be personal context, domain knowledge, or long-term working memory.

For smaller companies, it may be proprietary workflows, user-specific memory, or operational data that large foundation model providers cannot easily access.

If you do not have unique data at the beginning, memory is often the first practical moat.

When a user accumulates preferences and working state inside your agent, that memory starts becoming product differentiation.
