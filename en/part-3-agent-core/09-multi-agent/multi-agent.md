# 09. Multi-Agent

## Why Multi-Agent

In engineering, simplicity should remain the default.

So multi-agent is not automatically better. It only becomes worthwhile when dividing work actually reduces bottlenecks such as context pressure, waiting time, or task interference.

## How to Design Multi-Agent

A good multi-agent system starts with clear boundaries:

- what each agent owns
- what information is shared
- how tasks are handed off
- who makes final decisions

Without those boundaries, adding more agents often just multiplies complexity.
